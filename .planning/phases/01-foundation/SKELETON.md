# Walking Skeleton — Goblin vs. Superman

## What the Skeleton Delivers

The thinnest possible end-to-end working slice:
- Game window opens at 1280x800
- Background renders (cyan sky, 50px white ceiling, 50px green ground)
- All asset surfaces and sounds load from disk into a keyed cache
- Game loop runs at 60 FPS with delta-time
- Window closes cleanly on X button or Delete key

No gameplay. No players. No input. Just: it boots, it renders, it quits.

---

## Architectural Decisions

These decisions are locked for all subsequent phases. Changing them requires a new SKELETON.md or explicit decision record in CONTEXT.md.

### Runtime

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Language | Python 3.11 | pygame-ce 2.5.7 is installed for 3.11; PyInstaller 6.x tested on 3.11 |
| Game framework | pygame-ce 2.5.7 | Active fork of pygame; SDL2; identical API; mixer + display + clock |
| Class design | Plain classes (no pygame.sprite.Sprite) | Per CLAUDE.md constraint; simpler; update(dt)/draw(screen) interface |
| Test framework | pytest 9.0.3 | Pinned in requirements-dev.txt |

### Display

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Resolution | 1280x800 fixed | VB6 target; no pygame.SCALED (simpler, no interpolation artifacts) |
| Frame rate | 60 FPS via pygame.time.Clock | Single Clock.tick(60) loop; no set_timer for subsystems |
| Delta time | dt = clock.tick(60) / 1000.0 | Float seconds; all movement must multiply by dt |
| Window title | "Goblin vs. Superman" | Matches original VB6 game |

### Background Rendering

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Background approach | Pre-baked pygame.Surface | blit once per frame; no per-beam erasing (Pitfall 10 fix) |
| Sky color | RGB(0, 255, 255) — cyan | VB6 BackColor &H00FFFF00& decoded as OLE BGR |
| Ceiling band | 50px white at y=0..50 | Proportional translation from VB6 300-twip band |
| Ground band | 50px green at y=750..800 | Proportional translation from VB6 300-twip band |
| Play area | y=50 to y=750 — 700px tall | Between ceiling and ground |

### Asset Pipeline

| Decision | Choice | Rationale |
|----------|--------|-----------|
| ICO→PNG | Pillow 12.2.0 NEAREST resize | Only pure-Python tool that handles ICO AND-mask correctly |
| PNG size | 128x128 RGBA | 4x upscale from 32x32 source; preserves pixel art look |
| MIDI→WAV | FluidSynth CLI subprocess | Dev-tool only; not runtime; simpler than pyfluidsynth binding |
| Soundfont | tools/TimGM6mb.sf2 (~6MB, committed) | Free public-domain; no external download at build time |
| Source dir | raw_assets/ (committed to git) | Source of truth; committed ICO + WAV/MIDI files |
| Output dir | assets/ (gitignored) | Generated by convert_assets.py; never committed |

### Constants Architecture

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Single source of truth | settings.py | ALL geometry, color, speed, gameplay constants here. No hardcoding anywhere else. |
| Key constants | See table below | Derived from VB6 source and RESEARCH.md |

### Module Structure

| Module | Responsibility | Rule |
|--------|---------------|------|
| main.py | Bootstrap only: pygame.init(), display.set_mode(), create Game, call run(), pygame.quit() | No game logic here |
| game.py | Game loop, GameState enum, background baking, event dispatch | All timing owned here |
| assets.py | AssetCache class — load all surfaces/sounds at startup into keyed dicts | Load ONCE in __init__; never reload |
| settings.py | ALL constants — only file with hardcoded numbers | Other modules import from here |
| convert_assets.py | Dev-time: raw_assets/ → assets/ | Not imported by game; CLI only |

### Asset Cache Key Convention

All sprites and sounds are keyed by lowercase filename stem:
- GEvil.png → `cache.sprites["gevil"]`
- SUp1.png → `cache.sprites["sup1"]`
- LASER.WAV → `cache.sounds["laser"]`
- passport.wav → `cache.sounds["passport"]`

This convention is established in Phase 1 and must not change. Phase 2 (player.py) and Phase 3 (beam.py) consume assets using these exact keys.

### State Machine

GameState enum defined in game.py:
- `SPLASH` — intro screen (Phase 6)
- `PLAYING` — active game (Phase 2 onward)
- `PAUSED` — game paused (Phase 6)
- `GAME_OVER` — match ended (Phase 4)

Phase 1 skeleton starts in `PLAYING` state (no splash screen yet).

### VB6 Fidelity Decisions

| VB6 Behavior | Python Port Decision |
|-------------|---------------------|
| Per-sprite `LoadPicture()` in timers | Load once at startup in AssetCache.__init__() |
| Line() draw to erase beams | screen.blit(background_surface, (0,0)) each frame |
| Multiple VB6 Timers | Single Clock.tick(60) loop with dt |
| `or 4` beam direction check (VB6 bug) | Fixed: `beam.direction in (LEFT, RIGHT)` (Phase 3) |
| Twip-based coordinates | Fresh pixel constants in settings.py |
