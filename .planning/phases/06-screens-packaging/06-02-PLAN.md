## Context

The splash screen is a self-contained event loop. It plays canyon.wav music, cycles through random sprites every 750ms, advances credits one line every 1500ms, lets the player adjust the win score with Up/Down arrows, and returns the chosen win score when any other key or mouse click is pressed. Background color is dark navy `(0, 0, 64)` (VB6 BackColor D-06).

All 25 credits lines are verbatim from VB6 Form1.frm `strarrCredits` (D-04). Credits advance discretely — one line shown for 1500ms, then the next.

The function signature: `run_splash(screen: pygame.Surface, assets) -> int`
- `assets` is an `AssetCache` instance; use `assets.sprites` dict for the sprite pool.
- Returns the selected win score (int, default 50, range 10–500 in steps of 10).

### Task 2 — Implement `run_splash()` body

Inside `run_splash`:

**Music:** Start canyon.wav at splash entry:
```python
pygame.mixer.music.stop()
pygame.mixer.music.load(str(_MUSIC_PATH))
pygame.mixer.music.play(-1)
```

**Fonts:**
```python
font_title    = pygame.font.Font(None, 72)
font_sub      = pygame.font.Font(None, 36)
font_credits  = pygame.font.Font(None, 28)
font_winscore = pygame.font.Font(None, 28)
font_controls = pygame.font.Font(None, 22)
```

**Sprite pool:** Build at init, not per-frame:
```python
sprite_surfaces = list(assets.sprites.values())
current_sprite  = random.choice(sprite_surfaces)
sprite_timer    = 0.0
```

**Credits state:**
```python
credit_idx   = 0
credit_timer = 0.0
```

**Win score state:**
```python
win_score = WIN_SCORE   # default 50
```

**Clock:**
```python
clock = pygame.time.Clock()
```

**Main loop:**
```python
running = True
while running:
    dt = clock.tick(60) / 1000.0
    sprite_timer  += dt
    credit_timer  += dt

    if sprite_timer >= 0.75:
        sprite_timer -= 0.75
        current_sprite = random.choice(sprite_surfaces)

    if credit_timer >= 1.5:
        credit_timer -= 1.5
        credit_idx = (credit_idx + 1) % len(_CREDITS)

    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            pygame.quit()
            sys.exit()
        elif event.type == pygame.KEYDOWN:
            if event.key == pygame.K_DELETE:
                pygame.quit()
                sys.exit()
            elif event.key == pygame.K_UP:
                win_score = min(500, win_score + 10)
            elif event.key == pygame.K_DOWN:
                win_score = max(10, win_score - 10)
            else:
                running = False
        elif event.type == pygame.MOUSEBUTTONDOWN:
            running = False

    _draw(screen, font_title, font_sub, font_credits, font_winscore,
          font_controls, current_sprite, credit_idx, win_score)

return win_score
```

**Helper for shadow text (matches VB6 two-label shadow effect):**
```python
def _blit_centered(text, font, y, color=(255, 255, 255), shadow=(20, 20, 80)):
    shadow_surf = font.render(text, True, shadow)
    text_surf   = font.render(text, True, color)
    cx = SCREEN_W // 2
    screen.blit(shadow_surf, (cx - shadow_surf.get_width() // 2 + 2, y + 2))
    screen.blit(text_surf,   (cx - text_surf.get_width()   // 2,     y))
```

## Key Constraints

- `run_splash` must NOT import from `sound.py` — it calls `pygame.mixer.music` directly.
- `run_splash` must NOT import from `game.py` — no circular dependency.
- `_draw` must be a module-level function (not a closure inside `run_splash`) to keep `run_splash` readable; pass all arguments explicitly.
- Up/Down on splash adjust win score only — they do NOT dismiss the splash screen (D-09).
- Any other key or any mouse click dismisses the splash (D-04, D-09).
- `run_splash` returns an `int` (the selected win score). `main.py` passes this to `Game.__init__`.
