**Phase:** 6 — Screens & Packaging

**Goal:** Create `slappy.spec` and `build.bat` so `pyinstaller slappy.spec` produces a standalone Windows `.exe` in `dist/GoblinVsSuperman/` that bundles all assets and runs on a machine without Python installed.
**Requirements:** PKG-01

## Context

PyInstaller 6.x `--onedir` mode (COLLECT) places the exe and bundled files in `dist/GoblinVsSuperman/`:
```
dist/GoblinVsSuperman/
  GoblinVsSuperman.exe
  _internal/
    assets/sprites/*.png
    assets/sounds/*.wav
    main.pyc, game.pyc, splash.pyc, ...
    pygame/ (and other deps)
```

The existing `pathlib.Path(__file__).parent` pattern in `assets.py` and `sound.py` resolves to the `_internal/` directory in the bundle — this is where PyInstaller places module `.pyc` files and where `datas` are copied. **No code changes needed** for asset loading to work in the bundle.

`--windowed` suppresses the console window (Windows only). The `assets/` directory must be declared in `datas` so PyInstaller copies it into the bundle.

## Tasks

### Task 1 — Create `slappy.spec`

**File:** `slappy.spec` (project root)

```python
# -*- mode: python ; coding: utf-8 -*-

block_cipher = None

a = Analysis(
    ['main.py'],
    pathex=[],
    binaries=[],
    datas=[
        ('assets/sprites', 'assets/sprites'),
        ('assets/sounds',  'assets/sounds'),
    ],
    hiddenimports=[],
    hookspath=[],
    hooksconfig={},
    runtime_hooks=[],
    excludes=['tkinter', 'unittest', 'email', 'http', 'xml'],
    cipher=block_cipher,
    noarchive=False,
)

pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher)

exe = EXE(
    pyz,
    a.scripts,
    [],
    exclude_binaries=True,
    name='GoblinVsSuperman',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    console=False,
    disable_windowed_traceback=False,
    target_arch=None,
    codesign_identity=None,
    entitlements_file=None,
)

coll = COLLECT(
    exe,
    a.binaries,
    a.zipfiles,
    a.datas,
    strip=False,
    upx=True,
    upx_exclude=[],
    name='GoblinVsSuperman',
)
```

Key decisions:
- `datas` lists `sprites` and `sounds` subdirectories separately (not the whole `assets/` tree) so the bundle structure matches the load paths in `assets.py` and `sound.py`.
- `excludes` strips heavy stdlib modules not used by the game to reduce bundle size.
- `console=False` → windowed mode (no black terminal window).
- `upx=True` → UPX compression if UPX is installed; safe to leave enabled even if UPX is absent (PyInstaller skips silently).

### Task 2 — Create `build.bat`

**File:** `build.bat` (project root)

```bat
@echo off
echo Building GoblinVsSuperman...
pyinstaller slappy.spec --noconfirm
if %ERRORLEVEL% neq 0 (
    echo Build FAILED.
    exit /b 1
)
echo.
echo Build complete.
echo Executable: dist\GoblinVsSuperman\GoblinVsSuperman.exe
```

### Task 3 — Verify the spec runs without errors

Run the build to confirm the spec is syntactically valid and PyInstaller can analyze the project:

```
pyinstaller slappy.spec --noconfirm
```

Expected outcome: `dist/GoblinVsSuperman/GoblinVsSuperman.exe` is produced without analysis errors. Warnings about missing optional modules are acceptable; errors are not.

If the build fails with an `ImportError` about `pygame._sdl2` or similar optional pygame submodules, add them to `excludes` in the spec. If `assets/` is not found, verify the `datas` paths are relative to the project root (where `pyinstaller` is invoked).

### Task 4 — Smoke-test the bundle

After a successful build, launch the bundled exe:
```
dist\GoblinVsSuperman\GoblinVsSuperman.exe
```

Verify:
1. The window opens (no crash on launch)
2. The splash screen appears with the dark navy background
3. Music plays (or is present — may depend on audio driver on build machine)
4. Pressing a key enters the game

If the exe crashes at launch, run it with `--debug` flag or temporarily set `console=True` in the spec to capture the traceback, diagnose, fix, and rebuild.

## Key Constraints

- `datas` paths in the spec are relative to the directory where `pyinstaller` is invoked (project root). Do NOT use absolute paths.
- The `.gitignore` should exclude `dist/`, `build/`, and `*.spec` is kept (it is source). Verify `.gitignore` already covers `dist/` and `build/` — add if missing.
- `venv/` must NOT be inside the project root when running PyInstaller (or add `venv` to `excludes`/`pathex` carefully). Recommend running from the venv: `venv\Scripts\pyinstaller slappy.spec`.
- Do NOT commit the `dist/` or `build/` directories to git.
