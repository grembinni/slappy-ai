# Phase 6 Plan 01 — SoundManager: pause_game / resume_game

**Phase:** 6 — Screens & Packaging
**Plan:** 01 of 05
**Goal:** Add `pause_game()` and `resume_game()` methods to `SoundManager` so the PAUSED game state can freeze/unfreeze audio without disturbing the mute toggle.
**Requirements:** Enables UI-05
**Depends on:** Nothing (standalone `sound.py` change)
**Blocks:** 06-03-PLAN.md (game.py PAUSED state calls these)

---

## Context

`SoundManager.toggle_mute()` pauses/unpauses the mixer and sets `self._muted`. Phase 6 PAUSED state needs a separate pair of methods that freeze audio without touching `_muted`, so mute state is preserved across pauses (Decision D-03 in 06-CONTEXT.md).

Invariant: if the game is muted before pausing, the mixer is already silent — `pause_game()` must not double-unpause it on resume.

---

## Tasks

### Task 1 — Add `pause_game()` to SoundManager

**File:** `sound.py`
**After:** the closing line of `toggle_mute()`

```python
def pause_game(self) -> None:
    if not self._muted:
        pygame.mixer.pause()
        pygame.mixer.music.pause()
```

Guard on `not self._muted` ensures we only pause audio that is actually playing. If already muted (mixer already paused), this is a no-op.

### Task 2 — Add `resume_game()` to SoundManager

**File:** `sound.py`
**After:** `pause_game()`

```python
def resume_game(self) -> None:
    if not self._muted:
        pygame.mixer.unpause()
        pygame.mixer.music.unpause()
```

Guard on `not self._muted` ensures we don't accidentally restore audio when the player had muted before pausing.
